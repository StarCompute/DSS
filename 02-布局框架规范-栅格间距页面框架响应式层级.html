📐布局框架规范

栅格系统间距尺规页面框架响应式断点层级体系
📅 版本 v1.0 · 🔖 布局层 · 🏷️ 栅格间距响应式层级
本规范定义了系统的布局框架标准，包括24列栅格系统、8px基准间距体系、页面框架模板、响应式断点策略和z-index层级体系。所有布局组件均基于此规范构建，确保全系统视觉一致性。

📐一、栅格系统

核心原则：采用 24 列栅格 + 16px 水槽 + 24px 两侧留白（桌面端），基于 Flex 实现。 支持响应式前缀 .col-* / .col-md-* / .col-lg-*。

栅格规范定义

参数值说明
总列数24列24 可被 2、3、4、6、8、12 整除，组合灵活
水槽宽度16px列间距，通过负 margin + padding 实现
两侧留白(桌面)24px容器 padding，保证内容不贴边
两侧留白(平板)20px中屏适度缩小
两侧留白(手机)16px小屏紧凑但保留呼吸空间
实现方式Flexbox主流浏览器兼容性好，无需额外 polyfill

CSS 栅格实现

/* 24列栅格系统核心 CSS */
.row {
  display: flex;
  flex-wrap: wrap;
  margin-left: -8px; /* 负 gutter/2 */
  margin-right: -8px;
}
.col-6 { width: calc(100% / 24 * 6); padding: 0 8px; }
.col-8 { width: calc(100% / 24 * 8); padding: 0 8px; }
.col-12 { width: calc(100% / 24 * 12); padding: 0 8px; }
.col-16 { width: calc(100% / 24 * 16); padding: 0 8px; }

📏二、间距尺规

核心原则：所有外边距、内边距统一使用 8px 网格系统。 组件内用小尺度，区块间用中尺度，页面大模块间用大尺度。

间距使用指南

尺度值适用场景
微小4px, 8px图标与文字间距、标签内边距、表单字段间微调
小12px, 16px组件内部元素间距、卡片内边距、列表项间距
中24px, 32px区块之间间距、章节分隔、表单分组间距
大40px, 48px, 64px页面大模块之间、页面上下留白
✅ 最佳实践：使用 CSS 变量定义间距令牌（如 var(--spacing-4)），一处修改全局生效，杜绝散落不一致的硬编码数字。

🧱三、标准页面框架

核心原则：中后台系统采用经典的三区布局：顶部固定导航栏 + 左侧边栏 + 主内容区域（自适应）。 这是经过大量企业级项目验证的最稳定布局模式。

框架参数定义

区域规格说明
Header（顶部导航）高度 64px（桌面）/ 56px（移动端）position: sticky，背景半透明毛玻璃效果
Sidebar（侧边栏）展开 240px / 折叠 72px支持折叠切换，平板端自动折叠
Main（主内容）flex: 1 自适应内边距 24px，内容区最大宽度 1440px 居中

框架布局演示

🏷️ Header（64px · position: sticky）
⬅️ Sidebar（240px / 折叠72px）
📄 Main Content
自适应区域，内边距 24px，内容区最大宽度 1440px 居中
* 支持折叠侧边栏变 72px，移动端侧边栏默认隐藏（通过汉堡菜单唤起）

CSS 实现

/* 三区布局核心 */
.app-header {
  position: sticky; top: 0; z-index: 200;
  height: 64px; background: rgba(255,255,255,.92);
  backdrop-filter: blur(8px);
}
.app-layout { display: flex; }
.app-sidebar { width: 240px; flex-shrink: 0; }
.app-main { flex: 1; padding: 24px; }

📱四、响应式断点

核心原则：采用移动优先（Mobile First）策略，基于主流设备宽度定义 5 级断点。 从小屏开始写基础样式，通过 @media (min-width) 逐级增强。

断点定义

断点宽度设备典型适配
xs< 576px小手机单列布局、汉堡菜单
sm≥ 576px大手机双列、底部导航
md≥ 768px平板栅格自适应、侧边栏折叠
lg≥ 1024px桌面完整栅格、侧边栏展开
xl≥ 1280px大屏内容区最大 1440px 居中

断点场景演示

📱 手机
< 768px · 100%宽度 · 无侧边栏
📋 平板
768-1024px · 栅格自适应 · 侧边栏折叠
🖥️ 桌面
≥ 1024px · 完整栅格 · 展开侧边栏
🖵 大屏
≥ 1280px · max-width 1440px

响应式 CSS 写法

/* 移动优先：基础样式为手机 */
.col-24 { width: 100%; }

/* 平板增强 */
@media (min-width: 768px) {
  .col-md-8 { width: calc(100% / 24 * 8); }
}

/* 桌面增强 */
@media (min-width: 1024px) {
  .col-lg-6 { width: calc(100% / 24 * 6); }
}

🗂️五、层级体系（z-index）

核心原则：统一 z-index 层级标准，避免各组件随意设置导致的覆盖混乱。 层级从低到高：Base → 浮层 → 弹窗 → 全局提示。

层级定义表

层级z-index 值CSS 变量适用组件
Base0--z-base页面正常流内容
Elevated10--z-elevated卡片 hover 悬浮效果
Dropdown100--z-dropdown下拉菜单、选择器弹出层
Sticky200--z-sticky固定头部、吸顶导航
Modal Overlay1000--z-modal-overlay弹窗遮罩层
Modal1010--z-modal弹窗内容
Toast1200--z-toast全局消息提示
Tooltip1300--z-tooltip工具提示（最高优先级）

层级可视化

🔹 Base (0)
页面内容
🔸 Elevated (10)
卡片悬浮
🔹 Dropdown (100)
下拉菜单
🔸 Sticky (200)
固定头部
🔹 Overlay (1000)
遮罩层
🔸 Modal (1010)
弹窗内容
🔹 Toast (1200)
全局提示
🔸 Tooltip (1300)
工具提示
⚠️ 红线：禁止在组件中硬编码 z-index 数字（如 z-index: 9999），必须使用 CSS 变量引用层级令牌。避免出现 z-index 競賽。

✅七、布局框架 · 检查清单

✅ 栅格系统是否采用 24 列 + 16px 水槽？
✅ 两侧留白桌面端是否为 24px，移动端 16px？
✅ 所有间距是否使用 8px 倍数（4/8/12/16/24/32/40/48/64）？
✅ 页面框架是否包含 Header + Sidebar + Main 三区？
✅ z-index 是否使用 CSS 变量引用层级令牌，而非硬编码数字？
✅ 响应式断点是否覆盖 手机、平板、桌面、大屏 四级？
✅ 是否采用 移动优先（min-width）策略？
✅ 侧边栏是否支持 展开（240px）/ 折叠（72px）两种状态？
✅ 内容区最大宽度是否 ≤ 1440px 居中？
✅ 顶部导航是否使用 position: sticky 固定？
✅ 间距是否使用 CSS 变量（如 var(--spacing-4)）而非硬编码？
✅ 响应式列类是否提供 col-* / col-md-* / col-lg-* 三级前缀？
