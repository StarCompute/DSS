🔘 二、按钮配色方案规范

按钮是用户与系统交互的核心触点，其配色必须准确传达操作语义、引导用户行为、符合视觉层级。

2.1 按钮类型与配色总览

按钮类型背景色文字色边框悬停态典型场景
主要按钮 (Primary)#0066CC#FFFFFF无#0052A3提交、保存、确认、下一步
次要按钮 (Secondary)#FFFFFF#0066CC1px solid #D1D5DB背景 #F9FAFB取消、返回、暂存草稿
危险按钮 (Danger)#DC2626#FFFFFF无#B91C1C删除、注销、不可逆操作
成功按钮 (Success)#10B981#FFFFFF无#059669审批通过、确认完成、发布
警告按钮 (Warning)#F59E0B#FFFFFF无#D97706重置、强制操作、风险提示确认

2.2 设计理由详解

为什么用蓝色 #0066CC？

  • 品牌中性：蓝色是全球范围内最广泛接受的操作色，不受文化背景限制，传达"可信赖、可点击"的语义。
  • 对比度达标：#0066CC 在白色背景上对比度约 4.8:1，满足 WCAG AA 标准（≥4.5:1）。
  • 视觉层级：蓝色在页面中具有天然的"前进感"，能有效引导用户视线聚焦到主要操作上。
  • 避免冲突：红色留给错误/危险操作，绿色留给成功操作，蓝色作为"中性正向操作"不抢占语义色。
  • 悬停加深：#0052A3 比默认深约 10%，给予明确交互反馈而不过度跳跃。

取消按钮 —— 为什么是白底灰边框？

  • 视觉降权：取消是"非推荐路径"，使用次要样式使其在视觉上退后。
  • 认知一致性：用户已形成"灰/白 = 取消/返回"的心智模型。
  • 不推荐使用红色取消：红色应保留给"删除"等破坏性操作。

危险按钮 —— 为什么是红色 #DC2626？

  • 警示本能：红色在所有文化中都与"警告、危险、停止"关联。
  • 防止误操作：红色按钮会让用户在点击前产生额外的心理停顿，降低误触概率。
  • 二次确认：危险按钮通常需要配合二次确认弹窗使用，颜色 + 文案 + 弹窗构成三层防护。

按钮配色通用原则

原则说明
一页一个主按钮每个页面/表单区域最多一个 Primary 按钮，其余降级为 Secondary 或 Text 按钮
语义色不混用红色 ≠ 取消，绿色 ≠ 保存。语义色严格对应操作后果
对比度 ≥ 4.5:1按钮文字与背景的对比度必须满足 WCAG AA 标准
悬停态有反馈所有可点击按钮必须有 hover 状态变化，变化幅度控制在 10%-15%
焦点态可访问:focus-visible 添加 2px 轮廓，支持键盘导航用户

2.3 CSS 实现示例

/* ===== 按钮基础样式 ===== */
.btn { display:inline-flex; align-items:center; justify-content:center; height:40px; padding:0 20px; border-radius:8px; font-size:14px; font-weight:500; cursor:pointer; border:1px solid transparent; transition:all 0.15s; }
.btn-primary { background:#0066CC; color:#FFFFFF; }
.btn-primary:hover { background:#0052A3; }
.btn-secondary { background:#FFFFFF; color:#0066CC; border-color:#D1D5DB; }
.btn-secondary:hover { background:#F9FAFB; }
.btn-danger { background:#DC2626; color:#FFFFFF; }
.btn-danger:hover { background:#B91C1C; }
.btn:disabled { background:#E5E7EB; color:#9CA3AF; cursor:not-allowed; }

✅ 八、设计基础检查清单

  • 颜色使用语义化CSS变量，不硬编码色值
  • 按钮配色符合语义映射（蓝=操作、红=危险、绿=成功）
  • 字体使用系统默认字体栈，确保渲染性能
  • 字号梯度完整覆盖 12px~36px
  • 表格有表头背景色、悬停行高亮
  • Toast/Notification 统一使用深色底白字
  • 加载动画使用CSS animation，避免JS轮询
  • 表单必填项有红色星号标识
  • 错误提示位于输入框下方，红色12px
  • 所有交互元素有 :focus-visible 样式
