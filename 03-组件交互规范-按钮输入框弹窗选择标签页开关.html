🎨组件交互规范

按钮输入框弹窗选择器标签页开关尺寸一致性
📅 版本 v1.0 · 🔖 组件层 · 🏷️ 按钮输入弹窗选择标签开关
本规范定义了系统中所有基础交互组件的标准，包括按钮（主次链接文字图标）、输入框、弹窗、选择器、标签页和开关等。统一尺寸体系确保各组件之间的一致性。

🔘一、按钮 Button

核心原则：按钮是用户操作的主要触发点。统一三级尺寸（大48px / 中40px / 小32px）、三类样式（主要/次要/文字）， 含悬停、激活、禁用态的完整反馈。

按钮类型与尺寸

类型用途背景色文字色边框
主要按钮核心操作（提交、保存、确认）--primary#FFF无
次要按钮辅助操作（取消、返回）#FFF--primary1px solid #D1D5DB
文字按钮轻量操作（查看更多、编辑）透明--primary无

✏️二、输入框 Input

核心原则：输入框需要清晰的视觉状态区分——默认、悬停、聚焦、错误、禁用。 聚焦态使用品牌色边框 + 外发光环（box-shadow），错误态使用红色边框 + 错误提示文字。

☑️三、选择控件（Radio / Checkbox / Switch）

核心原则：单选（Radio）用于互斥选择，复选框（Checkbox）用于多选，开关（Switch）用于即时生效的二元状态切换。 三者视觉区分明确，不可混用。

<style>
.tabs { display: flex; border-bottom: 1px solid #E5E7EB; }
.tab { padding: 10px 20px; color: #6B7280; cursor: pointer; border-bottom: 2px solid transparent; }
.tab:hover { color: #111827; }
.tab.active { color: var(--primary); border-bottom-color: var(--primary); }
.tab-panel { padding: 24px 0; display: none; }
.tab-panel.active { display: block; }
</style>

<script>
// 标签页切换
document.querySelectorAll('.tab').forEach(tab => {
    tab.addEventListener('click', () => {
        const tabId = tab.getAttribute('data-tab');
        tab.closest('.card').querySelectorAll('.tab').forEach(t => t.classList.remove('active'));
        tab.classList.add('active');
        tab.closest('.card').querySelectorAll('.tab-panel').forEach(p => {
            p.classList.toggle('active', p.dataset.panel === tabId);
        });
    });
});
</script>

<style>
.select-wrapper { position: relative; width: 240px; }
.select-trigger { height: 40px; padding: 0 32px 0 12px; border: 1px solid #D1D5DB; border-radius: 6px; display: flex; align-items: center; justify-content: space-between; cursor: pointer; }
.select-dropdown { position: absolute; top: 46px; left: 0; right: 0; background: #fff; border-radius: 8px; box-shadow: 0 10px 15px -3px rgba(0,0,0,0.1); max-height: 200px; overflow-y: auto; display: none; }
.select-dropdown.show { display: block; }
.select-option { padding: 8px 12px; cursor: pointer; }
.select-option:hover { background: #F3F4F6; }
</style>

<script>
const trigger = document.querySelector('.select-trigger');
const dropdown = document.querySelector('.select-dropdown');
trigger.addEventListener('click', e => { e.stopPropagation(); dropdown.classList.toggle('show'); });
dropdown.querySelectorAll('.select-option').forEach(opt => {
    opt.addEventListener('click', function(e) {
        trigger.querySelector('span').textContent = this.textContent;
        dropdown.classList.remove('show');
        e.stopPropagation();
    });
});
document.addEventListener('click', () => dropdown.classList.remove('show'));
</script>

🪟七、弹窗 Modal / Dialog

核心原则：弹窗用于需要用户确认或输入信息的重要操作。 结构分为 Header（标题）、Body（内容）、Footer（操作按钮），背景半透明遮罩 + 毛玻璃效果。

规范参数

属性值说明
宽度560px（默认）/ max-width: 90vw移动端自适应
圆角16px大圆角柔和视觉
遮罩rgba(0,0,0,0.5) + backdrop-filter: blur(2px)半透明 + 毛玻璃增强层次感
入场动画scale(0.95) → scale(1)微缩放弹性效果

📐九、表单元素尺寸一致性规范

核心原则：同一行中的表单元素（输入框、下拉选择器、按钮）必须保持相同的视觉高度。 高度不一致会破坏对齐节奏，降低视觉质量。

四级高度标尺

级别高度适用场景示例
xs26px表格行内操作按钮查看 / 编辑 / 删除
sm32px筛选栏 / 工具栏 / 搜索条件区日期picker、下拉选择
md40px主表单 / 弹窗表单 / 主搜索框默认 input/select/button
lg48px首页大搜索框 / CTA 按钮首页搜索
⚠️ 红线：同一 flex-row / search-area / filter-row / toolbar 中的所有 input、select、button 必须使用同一级别的高度标尺值。禁止混用不同高度的元素。
❌ 错误：高度混用
input 40px
按钮 28px
↑ 高度差 12px，按钮「悬浮」
✅ 正确：统一高度
input 40px
按钮 40px
↑ 同一基线，视觉统一

CSS 写法规范

规范项❌ 禁止✅ 正确原因
按钮高度padding: 8px 20px;height: 40px; padding: 0 20px;
line-height: 1;		padding 撑开高度依赖字体，跨浏览器不一致
输入框高度padding: 8px 12px;height: 40px; padding: 0 12px;
box-sizing: border-box;		显式 height + box-sizing 确保一致
垂直对齐不设置对齐方式display: inline-flex;
align-items: center;		flex + center 确保文字垂直居中
定义方式硬编码数字height: var(--input-height-md);一处修改全局生效

✅十一、组件交互 · 检查清单

✅ 按钮是否定义了 三种尺寸（lg/md/sm）和三类样式（主要/次要/文字）？
✅ 按钮和输入框的尺寸是否使用 CSS 变量统一定义？
✅ 输入框是否包含默认、悬停、聚焦、错误、禁用五种状态的完整样式？
✅ 选择控件（Radio/Checkbox/Switch）是否视觉区分明确，不混用？
✅ 标签页激活态是否使用品牌色下划线 + 品牌色文字？
✅ 弹窗是否包含Header + Body + Footer 三区结构？
✅ 弹窗入场动画是否使用scale 弹性过渡？
✅ 同一行的 input、select、button 是否使用同一级别高度？
✅ 组件高度是否使用 height 显式声明（而非依赖 padding）？
✅ 所有颜色值是否通过 CSS 变量引用，避免硬编码？
✅ 聚焦态是否使用 box-shadow 外发光（而非 outline）？
✅ 禁用态是否有透明度 + cursor: not-allowed反馈？
确认操作
